# Hermes Agent 架构重构白皮书 (V1.0)

## 1. 核心问题诊断
- **模块耦合**: Gateway 直接持有 Agent 实例，导致 I/O 阻塞会影响生成逻辑。
- **环境污染**: 工具直接在 Host 路径运行，缺乏路径映射层，导致 Windows/MSYS 路径冲突。
- **状态不透明**: 锁残留和数据库死锁缺乏自动恢复机制。

## 2. 重构目标
- [ ] **解耦**: 引入事件驱动架构 (Event-Driven)。
- [ ] **容器化**: 工具执行环境虚拟化。
- [ ] **高可用**: 实现自愈式锁管理。
